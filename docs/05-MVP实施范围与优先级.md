# 呼和浩特城市信息网站：MVP实施范围与优先级

## 1. MVP目标（第一版上线）
- 在最短周期内上线“可读 + 可追溯”的城市信息网站。
- 优先满足：查看核心指标、阅读现状分析、查看预测区间、点击外部来源。
- 暂不追求复杂后台与高级交互，先保证内容可信和结构清晰。

## 2. 范围定义

### In Scope（MVP必须做）
- 页面：首页 + 5个专题页 + 数据来源页。
- 内容：每页至少 4 张核心图表 + 3 条关键结论 + 3 个外部链接。
- 引用：每个图表和关键结论均有来源脚注。
- 预测：至少覆盖人口、GDP、产业结构3类指标的双情景区间预测。
- 更新：提供手动更新流程文档（非自动化也可）。

### Out of Scope（MVP暂不做）
- 用户登录、评论、收藏等账户系统。
- 复杂BI交互（拖拽分析、多维钻取、地图联动）。
- 全自动爬取与ETL流水线。
- 移动端深度适配（先保证基础响应式）。

## 3. 实施优先级（P0/P1/P2）

### P0（上线阻断项）
- 信息架构与导航完成。
- 核心指标与口径统一完成。
- 来源映射表接入页面脚注。
- 首页、人口页、经济页先行可访问。

### P1（上线增强项）
- 产业页、教育页、规划页内容完善。
- 双情景预测图表上线（至少3个指标）。
- 数据来源页支持按“指标ID”筛选。

### P2（迭代优化项）
- 图表交互增强（年份切换、悬浮对比）。
- 自动更新脚本（按季度重算预测）。
- 数据下载（CSV/JSON）和更新日志可视化。

## 4. 建议里程碑（4周样例）
- 第1周：完成数据口径、来源映射、页面骨架。
- 第2周：完成首页 + 人口 + 经济页面内容与图表。
- 第3周：完成产业 + 教育 + 规划页面与预测模块。
- 第4周：完成来源页、全站校对、发布前验收。

## 5. MVP验收标准（可打勾）
- [ ] 全部页面可访问，无死链。
- [ ] 每条关键结论均可追溯到外部来源。
- [ ] 图表脚注包含来源、口径、更新时间。
- [ ] 预测结果包含基线/政策两情景及不确定性说明。
- [ ] 来源页可检索并包含至少20条权威来源记录。
- [ ] 内容描述与数据数值一致（抽检通过）。

## 6. 上线后首轮迭代建议
- 增加“城市对标”模块（与其他省会对比）。
- 增加“按区县查看”模块（若数据可得）。
- 增加“季度简报自动生成”功能。
